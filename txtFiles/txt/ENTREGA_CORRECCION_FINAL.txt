MAPEO DE DATOS CORREGIDO:
════════════════════════════════════════════════════════════════════════════════

PERMISOS (CM-TH-FR-003):
  PDF → Extractor → Controlador → Base de Datos

  cedula          → GN_TERCE.NUM_IDEN → resolverCodFunciPorCedula() → COD_FUNCI
  nombre          → PDF data           → Validación                → -
  motivo          → PDF data           → OBS_NOVED                → nvarchar
  fecha_novedad   → PDF data           → FEC_INI, FEC_FIN         → date
  hora_inicio     → PDF data           → OBS_NOVED (incluido)     → -
  hora_fin        → PDF data           → OBS_NOVED (incluido)     → -
  cantidad        → PDF data           → OBS_NOVED (incluido)     → -
  observaciones   → PDF data           → OBS_NOVED                → nvarchar

VACACIONES (CM-TH-SV-001):
  PDF → Extractor → Controlador → Base de Datos

  cedula          → GN_TERCE.NUM_IDEN → resolverCodFunciPorCedula() → COD_FUNCI
  fecha_inicio    → PDF data           → FEC_INI                  → date
  fecha_fin       → PDF data           → FEC_FIN                  → date
  cantidad        → PDF data           → DIAS_TOTAL               → int
  observaciones   → PDF data           → DIAGNOSTICO              → nvarchar


VALORES CALCULADOS AUTOMÁTICAMENTE:
════════════════════════════════════════════════════════════════════════════════

PARA PERMISOS:
  COD_NOVED       ← MAX(COD_NOVED) + 1 (auto-calculado)
  COD_CONC        ← LIKE '%PERMISO%' de NO_CONCE (buscado)
  COD_PERIOD      ← Período actual activo (buscado)
  FEC_REGI        ← CONVERT(date, GETDATE()) (auto)
  IND_APLICADO    ← 'N' (no aplicado)
  ACT_USUA        ← 'SISTEMA' (usuario)
  ACT_HORA        ← GETDATE() (auto)
  ACT_ESTA        ← 'A' (activo)

PARA VACACIONES:
  COD_NOVED       ← MAX(COD_NOVED) + 1 (auto-calculado)
  ACT_USUA        ← 'SISTEMA' (usuario)
  ACT_HORA        ← SYSDATETIME() (auto)
  ACT_ESTA        ← 'A' (activo)

FLUJO DE EJECUCIÓN ACTUALIZADO:
════════════════════════════════════════════════════════════════════════════════

1. Usuario sube PDF desde interfaz web (http://localhost:3000)
   ↓
2. Node.js multer recibe el archivo en memoria
   ↓
3. Se guarda temporalmente en carpeta /temp
   ↓
4. Se llama a python/procesar_pdf.py con ruta temporal
   ↓
5. Python ejecuta pdf_import_extension.py
   ├─ Detecta tipo (PERMISO o VACACIONES)
   ├─ Extrae todos los campos mediante OCR/text
   └─ Retorna JSON con datos completos
   ↓
6. Node.js recibe JSON con datosExtraidos
   ↓
7. Valida periodo actual en NO_PERIOD
   ├─ Si no hay período → ERROR (mostrado en interfaz)
   └─ Si hay → continúa
   ↓
8. Busca empleado en GN_TERCE por cedula (NUM_IDEN)
   ├─ Si no existe → ERROR (mostrado en interfaz)
   └─ Si existe → continúa
   ↓
9. Obtiene COD_FUNCI de GN_FUNCI
   ├─ Si no existe relación → ERROR (mostrado en interfaz)
   └─ Si existe → continúa
   ↓
10. Según tipo_novedad del PDF:

    ├─ SI ES PERMISO:
    │  ├─ Busca COD_CONC en NO_CONCE (LIKE '%PERMISO%')
    │  ├─ Calcula COD_NOVED = MAX + 1
    │  └─ Inserta en NO_NOVED con campos REALES ✓
    │
    └─ SI ES VACACIONES:
       ├─ Calcula COD_NOVED = MAX + 1
       └─ Inserta en NO_AUSEN con campos REALES ✓
    ↓
11. Si inserción exitosa:
    └─ Retorna { success: true, codNoved: X, mensaje: "..." }
    ↓
12. Node.js responde a interfaz con resultado
    └─ Status: INSERTADO (verde) + detalles
    ↓
13. Usuario ve confirmación en interfaz
