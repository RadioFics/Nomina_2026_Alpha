🎯 USO DE LA APLICACIÓN
═══════════════════════════════════════════════════════════════════════════════

PASO 1: Conectar a la Base de Datos
  • Ingresar servidor SQL (ej: SERVIDOR-NOMINA o 192.168.1.100)
  • Ingresar nombre BD: MineDax
  • Usuario y contraseña (opcional - usa Windows Auth si no se proporciona)
  • Hacer clic en "Conectar a BD"

PASO 2: Cargar Archivos PDF
  • Hacer clic en el área de carga
  • O arrastrar archivos PDF al área (drag & drop)
  • Sistema soporta múltiples archivos simultáneamente

PASO 3: Procesar Archivos
  • Hacer clic en "Procesar Archivos"
  • Sistema extrae datos automáticamente
  • Valida duplicados en BD
  • Inserta en tablas correctas (NO_NOVED, NO_AUSEN, NO_CONCE)

PASO 4: Ver Resultados
  • Tabla detallada con cada registro
  • Indicador de exitosos/errores
  • Exportar resultados como JSON o CSV


📊 QUÉ HACE EL SISTEMA
═══════════════════════════════════════════════════════════════════════════════

EXTRAE AUTOMÁTICAMENTE:
  ✓ Cédula del empleado (10 dígitos)
  ✓ Nombre completo
  ✓ Cargo y área
  ✓ Fechas (inicio, fin o fecha única)
  ✓ Horas o días
  ✓ Tipo de permiso/vacaciones
  ✓ Observaciones
  ✓ Información del jefe inmediato

VALIDA:
  ✓ Que no sea duplicado en BD
  ✓ Fechas superpuestas (vacaciones)
  ✓ Tipos de datos correctos
  ✓ Campos obligatorios

INSERTA EN:
  ✓ NO_NOVED (para permisos)
  ✓ NO_AUSEN (para vacaciones)
  ✓ NO_CONCE (para permisos remunerados)

📈 FLUJO DE DATOS
═══════════════════════════════════════════════════════════════════════════════

    ↓
MÓDULO 1: Extractor
├─ Detecta tipo de formulario
├─ Extrae datos con regex
└─ Retorna JSON con 10-14 campos

    ↓
MÓDULO 2: Validador
├─ Conecta a SQL Server
├─ Busca duplicados

    ↓
MÓDULO 3: Insertor
├─ Si no es duplicado → INSERTA

    ↓
MÓDULO 4: Interfaz
├─ Muestra resultado
├─ Tabla con estado de cada registro
└─ Opción de exportar

🚨 SOLUCIÓN DE PROBLEMAS
═══════════════════════════════════════════════════════════════════════════════

PROBLEMA: "ODBC Driver 17 for SQL Server not found"
SOLUCIÓN: Instalar desde:
https://learn.microsoft.com/en-us/sql/connect/odbc/download-odbc-driver-for-sql-server

PROBLEMA: "Connection refused"
SOLUCIÓN: Verificar:
  • Servidor SQL está corriendo
  • Nombre del servidor es correcto
  • Port 1433 está disponible

PROBLEMA: "Authentication failed"
SOLUCIÓN: Verificar credenciales o usar autenticación Windows

PROBLEMA: "PDF no se procesa"
SOLUCIÓN: Verificar que sea un formulario válido (CM-TH-FR-003 o CM-TH-SV-001)
